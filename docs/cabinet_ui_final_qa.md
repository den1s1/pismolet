# Финальный UI QA личного кабинета

Дата: 2026-06-26

Статус: Sprint 8, краткий QA-отчёт по результатам UI-спринтов 1-7.

## Проверенный пользовательский сценарий

Целевой сценарий:

1. регистрация и вход;
2. создание рассылки;
3. загрузка или ручной ввод адресов;
4. подтверждение базы и типа письма;
5. написание письма и проверка preview;
6. расчёт и оплата;
7. запуск отправки;
8. возврат в историю и открытие отчёта.

Проверка зафиксирована через endpoint/smoke/css-contract тесты и ревью изменений. Полноценные screenshots не прикладывались; для передрелизного просмотра остаётся пройти те же экраны вручную в браузере на desktop, 768px и 390px.

## Что осталось за рамками UI-спринтов

- Полный visual regression со screenshots в браузере: desktop, 768px, 390px.
- Фактическая проверка touch/keyboard поведения на мобильном устройстве.
- Дальнейшее снижение `font-weight: 800/900` в админке и технических таблицах, если после ручного просмотра останется визуальный шум.
- Отдельная полировка старых одиночных страниц, которые не входят в основной wizard-путь.

## Рекомендованный ручной чек-лист перед релизом

1. Регистрация: два согласия, чекбоксы не отрываются от текста.
2. Dashboard: история читается, primary-действие не конкурирует с меню.
3. Адреса: загрузка, warning, поиск, сброс, добавление, удаление.
4. Письмо: preview без служебной техники, details раскрывается вручную.
5. Оплата: один финальный checkbox, сумма и расчёт понятны.
6. Запуск: один главный CTA, подробный отчёт закрыт по умолчанию.
7. Mobile 390px: нет горизонтального скролла вне `.table-wrap`.
