# SEO-аудит запуска продакшена Письмолёта

Дата: 2026-06-26  
Раздел: SEO и маркетинг  
Статус: первичный аудит прод-кода выполнен, внешняя проверка live HTTP-статусов требует подтверждения из браузера/сервера

## 1. Объём проверки

Проверка относится к публичному сайту `pismolet.ru` и статическим файлам в `public_html` ветки `Development`.

Проверены:

- структура URL из `sitemap.xml`;
- `robots.txt`;
- главная страница;
- тарифы;
- контакты и юридический раздел в рамках ранее подготовленной структуры;
- наличие базовых `title`, `description`, `canonical`;
- отсутствие очевидных внутренних и тестовых формулировок;
- отсутствие публичных payment success/fail страниц в `public_html`;
- базовая связка публичного сайта с `app.pismolet.ru` через кнопки входа и регистрации.

## 2. Статус robots.txt

`public_html/robots.txt` открыт для индексации:

```text
User-agent: *
Allow: /

Sitemap: https://pismolet.ru/sitemap.xml
```

Решение: оставлять открытым для продакшена.

## 3. Статус sitemap.xml

`sitemap.xml` содержит основные публичные страницы:

- `/`;
- `/how-it-works/`;
- `/pricing/`;
- `/anti-spam/`;
- `/contacts/`;
- `/legal/`;
- `/legal/offer/`;
- `/legal/privacy/`;
- `/legal/rules/`;
- `/legal/refund/`;
- `/legal/data-processing/`;
- `/articles/`;
- 4 опубликованные статьи;
- 6 SEO-посадочных страниц.

Публичные страницы оплаты `/payment/success/` и `/payment/fail/` в sitemap отсутствуют, потому что платёжные URL должны быть реализованы на стороне `app.pismolet.ru`.

## 4. Главная страница

На главной странице есть:

- корректный `title`;
- `description`;
- canonical на `https://pismolet.ru/`;
- базовые OpenGraph-теги;
- ссылки на вход и регистрацию в приложении;
- единое верхнее меню;
- ссылка на favicon;
- подключение общего CSS с cache-busting параметром.

Рекомендация: позже добавить OpenGraph не только на главную, но и на основные посадочные страницы и статьи.

## 5. Страница тарифов

На `/pricing/` зафиксированы:

- `title`;
- `description`;
- canonical;
- финальная тарифная сетка;
- объяснение модели оплаты;
- ссылка на условия возврата и оферту;
- CTA на регистрацию.

Тарифы на странице совпадают с текущим продуктовым решением:

- до 299 писем — 1 ₽ / письмо;
- 300–499 писем — 0,90 ₽ / письмо;
- 500–999 писем — 0,80 ₽ / письмо;
- 1000 и более писем — 0,70 ₽ / письмо.

## 6. Стоп-слова и признаки незавершённости

Критичных совпадений не найдено.

## 7. Что не удалось проверить из среды чата

Из среды чата не удалось подтвердить live HTTP-статусы сайта из-за недоступности DNS/внешнего fetch для домена `pismolet.ru`.

Нужно вручную или с сервера выполнить:

```bash
curl -I https://pismolet.ru/
curl -I https://pismolet.ru/robots.txt
curl -I https://pismolet.ru/sitemap.xml
curl -I https://pismolet.ru/pricing/
curl -I https://pismolet.ru/contacts/
curl -I https://pismolet.ru/legal/
curl -I https://pismolet.ru/articles/
```

Ожидаемо:

- `200 OK` для основных страниц;
- `Content-Type: text/html` для HTML-страниц;
- `Content-Type: text/plain` или близкий для `robots.txt`;
- `Content-Type: application/xml`, `text/xml` или близкий для `sitemap.xml`;
- отсутствие редиректов с `https` на `http`.

## 8. Ручные внешние действия после подтверждения live-статусов

1. Добавить сайт в Яндекс.Вебмастер.
2. Добавить сайт в Google Search Console.
3. Подтвердить права на домен или сайт.
4. Отправить `https://pismolet.ru/sitemap.xml` на переобход.
5. Проверить, что `app.pismolet.ru` не индексируется, если приложение пока не должно попадать в поиск.
6. После индексации проверить брендовый запрос `Письмолёт` и основные посадочные URL.

## 9. Найденные SEO-долги

Не блокируют запуск, но нужны следующим этапом:

1. Добавить OpenGraph/Twitter Card на все основные страницы, статьи и посадочные.
2. Добавить JSON-LD: `Organization`, `WebSite`, `Article`, `BreadcrumbList`, при необходимости `FAQPage`.
3. Провести браузерную проверку мобильной версии после деплоя.
4. Проверить фактические 404/редиректы live-сайта.
5. Проверить индексацию `app.pismolet.ru` и при необходимости закрыть её отдельным `robots.txt`/noindex.

## 10. Итог

Первичный SEO-аудит продакшен-кода пройден. Критичных SEO-блокеров в статической части сайта не найдено. Перед отправкой sitemap в поисковые системы нужно подтвердить live HTTP-статусы и доступность страниц уже на опубликованном домене.
