# Open tracking rollout plan

Документ фиксирует план внедрения аналитики открытий писем для Pismolet.

Важно: в интерфейсе и документации используем термин **Открыто**, а не **Прочитано**. Open tracking показывает факт загрузки tracking pixel и не является точным доказательством чтения письма получателем.

## Статус на 22 июня 2026 г.

- Sprint 5.2 - закрыт: backend фиксирует открытия через `GET /t/open/{token}.gif`, данные сохраняются в `send_events`.
- Sprint 5.3 - закрыт: tracking pixel вставляется в HTML-письма, тестовая рассылка на 2 адреса доставлена не в спам, открытия зафиксированы в БД.
- Sprint 5.4 - закрыт: открытия показаны в ЛК и в админской карточке кампании.
- Sprint 5.5.1 - закрыт: backend click tracking, SQL-скрипт и endpoint `GET /t/click/{token}` проверены и задеплоены.
- Sprint 5.5.2 - на проверке: абсолютные `http/https` ссылки в HTML-письме переписываются на `/t/click/{token}`; unsubscribe, open pixel, уже отслеживаемые ссылки и не-http ссылки не трогаются.

Подтверждение Sprint 5.3:

- новые письма получают `TrackingToken`;
- HTML содержит ссылку `https://app.pismolet.ru/t/open/`;
- после открытия писем заполнены `FirstOpenedAt`, `LastOpenedAt`, `OpenCount = 1`;
- endpoint возвращает `image/gif`, прозрачный GIF 1x1.

## Sprint 5.2 - Open tracking backend

Статус: **закрыт**.

Цель: начать фиксировать открытия на backend-уровне.

Входит:

- добавить поля в `send_events`:
  - `TrackingToken` - уникальный непрозрачный токен;
  - `FirstOpenedAt` - первое зафиксированное открытие;
  - `LastOpenedAt` - последнее зафиксированное открытие;
  - `OpenCount` - количество зафиксированных открытий;
- генерировать `TrackingToken` при создании `SendEvent`;
- добавить endpoint `GET /t/open/{token}.gif`;
- endpoint должен всегда возвращать прозрачную картинку, даже если token неизвестен;
- при валидном token:
  - если `FirstOpenedAt` пустой, заполнить;
  - обновить `LastOpenedAt`;
  - увеличить `OpenCount`;
- добавить backend-тесты.

Результат Sprint 5.2: открытия можно зафиксировать напрямую через endpoint, данные сохраняются в БД.

## Sprint 5.3 - Open tracking in emails

Статус: **закрыт**.

Цель: реально вставлять tracking pixel в HTML-письма.

Входит:

- добавлять в HTML-письмо пиксель вида `/t/open/{trackingToken}.gif`;
- строить абсолютный URL через `App__PublicBaseUrl`;
- не вставлять tracking pixel в plain text;
- не ломать unsubscribe block и текущую SMTP-отправку;
- проверить Gmail, Yandex, Mail.ru.

Результат Sprint 5.3: открытие HTML-письма создаёт open event.

## Sprint 5.4 - UI analytics

Статус: **закрыт**.

Цель: показать открытия пользователю и администратору.

Входит:

- добавить в список получателей поля:
  - `Открыто`;
  - `Первое открытие`;
  - `Последнее открытие`;
  - `Открытий`;
- добавить сводку по кампании:
  - отправлено;
  - принято SMTP;
  - открыто;
  - отписались;
  - ошибки;
- использовать только формулировки `Открыто`, `Открытия`, `Первое открытие`, `Последнее открытие`.

Результат Sprint 5.4: пользователь и администратор видят базовую аналитику открытий.

## Sprint 5.5.1 - Click tracking backend

Статус: **закрыт**.

Цель: подготовить backend для фиксации переходов по ссылкам без переписывания ссылок в реальных письмах.

Входит:

- добавить таблицу `tracked_links` для зарегистрированных ссылок;
- добавить таблицу `click_events` для событий перехода;
- добавить endpoint `GET /t/click/{token}`;
- при валидном token:
  - увеличить `ClickCount`;
  - заполнить `FirstClickedAt`, если это первый клик;
  - обновить `LastClickedAt`;
  - записать событие в `click_events`;
  - сделать redirect на исходную ссылку;
- неизвестный token должен возвращать `404`, а не редирект;
- добавить backend-тесты.

Результат Sprint 5.5.1: backend умеет безопасно фиксировать переходы и редиректить по заранее зарегистрированным ссылкам.

## Sprint 5.5.2 - Click tracking in emails

Статус: **на проверке**.

Цель: переписывать обычные абсолютные ссылки в HTML-письмах на tracking redirect.

Входит:

- находить абсолютные `http/https` ссылки в HTML-версии письма;
- создавать или переиспользовать запись в `tracked_links`;
- менять `href` на `/t/click/{token}`;
- оставлять видимый текст ссылки прежним;
- не трогать:
  - unsubscribe;
  - `/t/open/`;
  - уже отслеживаемые `/t/click/`;
  - `mailto:`;
  - `tel:`;
  - якоря;
  - относительные ссылки;
- добавить тесты на исключения.

Результат Sprint 5.5.2 после проверки: переход по ссылке из HTML-письма должен записываться в `click_events` и редиректить на исходный URL.

## После open tracking

Следующие отдельные фичи:

- click tracking UI analytics;
- bounce parser для возвратов;
- Postfix log parser для delivered/rejected;
- seed inbox monitor для проверки Inbox/Spam на контрольных ящиках.

## Ограничения

Open tracking не гарантирует точное чтение письма:

- почтовые клиенты могут кешировать картинки;
- изображения могут быть отключены;
- предпросмотр может считаться открытием;
- Apple Mail и proxy/privacy-защита могут искажать статистику;
- plain text письма не дают открытия через pixel.
