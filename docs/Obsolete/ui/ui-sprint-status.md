# Статус UI-спринтов

Дата фиксации: 2026-06-21.
Ветка: `Development`.
Статус документа: UI freeze MVP.

## Текущий статус

UI-цикл MVP закрыт. Спринты 0-12 приняты по build/test gate. Последняя проверка на сервере подтверждена пользователем как зелёная:

```text
dotnet build Pismolet.sln /nr:false -m:1 - passed
dotnet test Pismolet.sln --no-build - passed
```

Старые warning по MailKit advisory и obsolete Hangfire API остаются известным неблокирующим шумом.

## Что готово в UI

### Личный кабинет

- Dashboard с историей рассылок и состояниями кампаний.
- Профиль пользователя.
- Экран платежей пользователя.
- Wizard создания рассылки:
  - шаг 1 - адреса;
  - шаг 2 - письмо и превью;
  - шаг 3 - расчёт и оплата;
  - запуск, очередь и история.

### Админка

- Общий admin layout и навигация.
- `/admin/users` - список пользователей.
- `/admin/users/{email}` - профиль пользователя.
- `/admin/recipients` - получатели и глобальные статусы.
- `/admin/recipients/{email}` - профиль получателя.
- `/admin/campaigns` - список кампаний.
- `/admin/campaigns/{campaignId}` - профиль кампании, действия модерации и отправки.
- `/admin/payments` - список оплат и фильтры.
- `/admin/payments/{campaignId}` - профиль платежа и ручная сверка.
- `/admin/settings` - настройки сервиса: биллинг, лимиты, модерация, SMTP, системные параметры, admin allowlist.
- Регрессионные тесты основных admin-маршрутов и защиты от доступа не-админов.

## Что важно помнить после UI freeze

1. Админка закрыта через allowlist. На сервере должен быть задан `Admin__AllowedEmails` или совместимый ключ конфигурации.
2. `Mailing.Recipients` содержит только адресатов, принятых к отправке. Исключённые строки проверяются через `LastImportStats` и `LastImportBatch.Issues`.
3. Тестовый запуск должен использовать `Testing`/InMemory/FakeMailer и не ходить в production Postgres.
4. Следующие backend-зоны ещё не являются полноценной боевой реализацией:
   - постоянное хранилище настроек из `/admin/settings`;
   - реальный биллинг и сверка оплат;
   - реальный SMTP-test из админки;
   - полноценный risk scoring и модерационные правила;
   - собственный SMTP-сервер варианта Б.
5. После каждого следующего backend-спринта обязательный gate:

```bash
dotnet build Pismolet.sln /nr:false -m:1
dotnet test Pismolet.sln --no-build
```
