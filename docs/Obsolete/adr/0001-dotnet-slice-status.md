# ADR 0001 — статус текущего .NET-среза

Дата: 2026-06-16  
Статус: принято как рабочее решение для ветки `Development`, требуется синхронизация основного ТЗ

## Контекст

В ветке `Development` уже существует первый вертикальный срез приложения на .NET / ASP.NET Core:

- `src/Pismolet.Web`;
- cookie-аутентификация;
- регистрация, подтверждение email, вход, выход;
- личный кабинет;
- fake mailer;
- in-memory repository и audit log.

При этом `docs/platform_tz.md` исторически описывает MVP-стек как Python / Django / PostgreSQL / Redis / Celery. `docs/architecture_dotnet.md` и `docs/sprints.md` уже фиксируют .NET-направление для MVP.

## Решение

Для ветки `Development` текущий .NET / ASP.NET Core срез считается рабочей основой дальнейшей разработки MVP.

Sprint 0 не создаёт новый Django-проект и не развивает два стека одновременно.

До отдельной синхронизации `docs/platform_tz.md`:

- новые задачи разработки выполняются поверх `src/Pismolet.Web`;
- инфраструктурные production-замены in-memory реализаций планируются отдельными задачами;
- PostgreSQL, EF Core, Hangfire и реальные email/payment-провайдеры не подключаются без отдельного решения.

## Последствия

Плюсы:

- сохраняется уже созданный вертикальный срез;
- Sprint 1 можно продолжать как ревизию и доработку существующего кода;
- LLM-агентам проще ориентироваться по фактической структуре проекта.

Минусы и риски:

- `docs/platform_tz.md` временно противоречит фактическому коду;
- текущая in-memory реализация не является production-архитектурой;
- `dev:` password hash должен быть заменён до MVP/production.

## Требуется синхронизация

Нужно обновить `docs/platform_tz.md` одним из вариантов:

1. подтвердить переход MVP на .NET / ASP.NET Core;
2. признать текущий .NET-код прототипом и запланировать переписывание;
3. временно оставить .NET-срез экспериментом, но не развивать параллельно Django.

До принятия обратного решения основной путь в ветке `Development` — .NET.
