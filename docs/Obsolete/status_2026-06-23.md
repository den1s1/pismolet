# Статус разработки на 2026-06-23

## Контекст

Работа ведется в ветке `Development` проекта `den1s1/pismolet`.

Этот файл фиксирует состояние после production-проверок блока доставляемости, suppression, UI-отчетов, экспортов и закрытия технических warning-ов сборки.

## Подтверждено в production

### 1. HardBounce suppression

Статус: подтверждено.

Проверено:

- Postfix permanent bounce мапится в `send_events.DeliveryStatus = HardBounce`.
- Адрес с `HardBounce` добавляется в `client_suppressions`.
- Suppression привязан к клиенту по `ClientId = OwnerEmail`.
- В `client_suppressions` сохраняются:
  - `EmailNormalized`;
  - `Reason = HardBounce`;
  - `SourceMailingId`;
  - `SourceProviderMessageId`;
  - `CreatedAt`;
  - `LastSeenAt`.

Production-пример:

- email: `infa@stranachitaliya.ru`;
- delivery status: `HardBounce`;
- provider message id: `0049A842B1`;
- client: `den1s@mail.ru`.

### 2. UI suppression на импорте

Статус: подтверждено.

Проверено:

- адрес из `client_suppressions` исключается уже на этапе импорта;
- пользователь видит полный email без маскирования;
- причина отображается как ошибка доставки;
- исключенный адрес не попадает в оплату и отправку.

Production-пример:

- 2 строки в файле;
- 1 адрес принят к отправке;
- 1 адрес исключен из-за ошибки доставки;
- исключенный адрес не оплачивается.

### 3. Убрано маскирование email в UI

Статус: подтверждено.

Проверено:

- в импорте показывается полный email;
- в оплате показывается полный email;
- в отчете отправки показывается полный email;
- звезды вида `i***@domain.ru` больше не используются в пользовательском UI отправки.

### 4. Excel-экспорт отчета рассылки

Статус: подтверждено.

Добавлено:

- основной экспорт `.xlsx`;
- запасной экспорт `.csv`;
- кнопка в UI отправки: `Скачать Excel-отчет`.

Excel-отчет открывается сразу по колонкам и не требует ручного разделения CSV.

Колонки отчета:

- Email;
- Статус отправки;
- Статус доставки;
- Открытий;
- Первое открытие UTC;
- Последнее открытие UTC;
- Кликов;
- Первый клик UTC;
- Последний клик UTC;
- Причина ошибки доставки.

Дополнительно исправлено поведение причины ошибки:

- для `Delivered` технический SMTP-ответ вида `dsn=2.0.0 / 250 OK` не считается ошибкой и не выводится в колонку причины ошибки доставки;
- для `HardBounce`, `SoftBounce`, `Rejected`, `Failed`, `Skipped`, `Paused` причина сохраняется в отчете.

### 5. SoftBounce накопительный анализ

Статус: подтверждено.

Реализован безопасный вариант без блокировки адреса.

Поведение:

- система считает накопленные `SoftBounce` по паре клиент + email;
- порог предупреждения: 2 временные ошибки;
- адрес не попадает в suppression;
- адрес не исключается;
- адрес остается принят к отправке;
- пользователь видит предупреждение на импорте.

Production-проверка выполнена на тестовом адресе:

- email: `softbounce-check@example.test`;
- client: `den1s@mail.ru`;
- soft bounce count: 2.

Ожидаемое и подтвержденное UI-сообщение:

```text
Временные ошибки доставки ранее: 2. Адрес не исключен.
```

Счетчики на импорте:

- строк в файле: 1;
- принято к отправке: 1;
- дублей и ошибок: 0;
- не сможем отправить: 0;
- ранее отписались: 0.

### 6. UX-разделение предупреждений и исключений

Статус: подтверждено в production.

Проверено:

- накопленные `SoftBounce` больше не отображаются внутри блока `Что исключено`;
- для адресов, которые приняты к отправке, но имеют накопленные временные ошибки, используется отдельный блок `Предупреждения`;
- блок `Что исключено` показывает только реально исключенные адреса;
- если исключенных адресов нет, пользователь видит `Исключенных адресов нет.`.

Production-проверка выполнена на `softbounce-check@example.test`.

### 7. Админский обзор доставки и suppression

Статус: подтверждено в production.

Добавлена страница:

- страница открывается без HTTP 500;

- отображаются счетчики событий доставки за период;
- отображаются `Delivered`, `HardBounce`, `SoftBounce`, `Rejected`, `Complaint`;

- отображается общее количество `client_suppressions` в базе;
- отображается таблица клиентов с проблемами доставки;
- отображается таблица клиентов с suppression за период;
- production-значения видны для клиента `den1s@mail.ru`.
