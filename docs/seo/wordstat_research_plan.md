# План работы с Яндекс Wordstat для Письмолёта

Дата: 2026-06-26  
Раздел: SEO и маркетинг  
Статус: подготовлена рабочая заготовка для сбора семантики; нужны реальные выгрузки частот из Wordstat

## 1. Цель

Собрать поисковую семантику для Письмолёта и разложить её по страницам сайта:

- главная;
- посадочные страницы;
- статьи;
- будущие статьи;
- возможные страницы сравнения;
- рекламные группы для Яндекс Директа.

Фокус продукта: простой сервис email-рассылок для малого бизнеса, НКО, мероприятий и пользователей без технической подготовки.

## 2. Важные ограничения позиционирования

Не собирать и не использовать как целевые группы запросы, которые тянут холодный спам, незаконные базы или обход ограничений.

Запрещённые или нежелательные направления:

- купить email базу;
- база email адресов скачать;
- спам рассылка;
- массовая рассылка без согласия;
- холодная рассылка по чужой базе;
- парсинг email;
- рассылка по купленной базе;
- обойти блокировку Gmail/Яндекс;
- как отправить 10000 писем без блокировки.

Такие запросы можно фиксировать как минус-слова или как антиспам-контент только в образовательном контексте, без привлечения этой аудитории.

## 3. Регионы для первичной проверки

Первичный регион: Россия.

Дополнительно посмотреть:

- Москва и область;
- Санкт-Петербург и Ленинградская область;
- крупные города РФ;
- без региона, если Wordstat показывает заметную разницу.

## 6. Как выгружать из Wordstat

Для каждой стартовой фразы:

1. Вбить фразу в Wordstat.
2. Выбрать регион `Россия`.
3. Зафиксировать:
   - базовую частотность;
   - похожие запросы из правой колонки;
   - вложенные формулировки из левой колонки;
   - сезонность, если заметна.
4. Повторить для Москвы и Санкт-Петербурга по самым важным группам.
5. Выгрузить в таблицу.

Рекомендуемые колонки таблицы:

| Запрос | Частотность РФ | Частотность Москва | Частотность СПб | Группа | Интент | Целевая страница | Приоритет | Комментарий |
|---|---:|---:|---:|---|---|---|---|---|

Интенты:

- коммерческий;
- информационный;
- навигационный/брендовый;
- проблемный;
- нежелательный/антиспам.

## 8. Первые гипотезы для новых страниц

После Wordstat стоит проверить необходимость страниц:

1. `/school-email-list/` — рассылка для школы, кружка или секции.
2. `/event-email-reminders/` — напоминания участникам мероприятия.
3. `/email-list-cleanup/` — подготовка и чистка списка email.
4. `/unsubscribe-link/` — ссылка отписки в рассылке.
5. `/gmail-bcc-limits/` — ограничения скрытой копии в обычной почте.
6. `/email-newsletter-small-team/` — рассылка для небольшой команды или проекта.

Создавать страницы только после подтверждения спроса и самостоятельного интента.

## 9. Что делать после выгрузки

1. Удалить мусорные и опасные запросы.
2. Разложить оставшиеся запросы по интентам.
3. Сравнить с текущей структурой сайта.
4. Найти страницы, где уже есть релевантный URL, но не хватает текста/метатегов.
5. Найти запросы, под которые нужна новая статья или посадочная.
6. Обновить `docs/site_production_plan.md` и контент-план.
7. После публикации отправить новые URL в Яндекс.Вебмастер и Google Search Console.

## 10. Приоритет на первый проход

Сначала проверить не всё подряд, а 5 групп:

1. `письмолёт` / `письмолет` / `pismolet`.
2. `рассылка через скрытую копию` и смежные BCC-запросы.
3. `рассылка из excel` и смежные запросы.
4. `email рассылка без настройки домена` и смежные запросы.
5. `рассылка участникам мероприятия` и `email рассылка для малого бизнеса`.

Эти группы ближе всего к текущему позиционированию и уже имеют посадочные страницы на сайте.
