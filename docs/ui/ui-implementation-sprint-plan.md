# План спринтов: доведение UI ЛК и админки до макетов

Дата фиксации: 2026-06-21.

Основа:

- `ui/mvp-flow.html` - эскиз личного кабинета пользователя и сценария запуска рассылки.
- `ui/admin-cabinet.html` - эскиз административного кабинета.
- Текущая реализация приложения - ASP.NET Core, server-rendered HTML через `src/Pismolet.Web/Rendering/HtmlRenderer.cs`.

Цель плана - поэтапно привести рабочий интерфейс Письмолёта к состоянию, заложенному в HTML-эскизах, без попытки за один раз переписать весь UI.

## Общие принципы реализации

1. Идём от работающих сценариев к визуальному усложнению.
2. Сначала переносим дизайн-систему и общий каркас, потом отдельные экраны.
3. Каждый спринт должен завершаться деплоем и визуальной проверкой на `https://app.pismolet.ru`.
4. Макеты в `ui/*.html` считаются визуальным эталоном, но реальные данные берём из текущей доменной модели.
5. В MVP не вводим лишний JavaScript-фреймворк. Базовый путь - продолжать server-rendered HTML, вынести общие UI-компоненты и CSS.
6. Везде сохраняем обязательные элементы рассылки: служебный блок, глобальная отписка, подтверждение законности базы, исключение отписавшихся адресов.

   - `/account/register`
2. Сопоставить текущие экраны с макетами:
   - ЛК пользователя из `ui/mvp-flow.html`.
   - Админка из `ui/admin-cabinet.html`.
3. Найти текущие места генерации HTML в `HtmlRenderer.cs`.
4. Разделить рендеринг на логические блоки:
   - layout;
   - header;
   - buttons;
   - cards;
   - forms;
   - tables;
   - badges;
   - mailing wizard;
   - admin screens.
5. Завести единый CSS-файл или единый CSS-блок для приложения.
6. Зафиксировать визуальные токены из макетов:
   - цвет фона;

```text

Общие стили вынесены или подготовлены к вынесению.
Внешний вид пока может быть старым, но структура готова к переносу.
```

## Примечания для реализации
