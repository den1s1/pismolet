# Sprint 7 — ручные проверки глобальной отписки

## Happy path

1. Запустить приложение в `Development`.
2. Зарегистрировать/войти тестовым клиентом.
3. Создать рассылку.
4. Загрузить CSV с валидным адресом `ok@example.test`.
5. Подтвердить базу.
6. Сохранить plain text письмо.
7. Пройти fake-оплату.
8. Пройти проверку перед отправкой до статуса «Одобрено».
9. Открыть `/mailings/{id}/send` и запустить отправку.
10. Открыть `/dev/fake-mailer`.
11. Найти fake-письмо для `ok@example.test`.
12. Открыть unsubscribe link без авторизации.
13. Нажать «Отписаться».
14. Убедиться, что страница показывает спокойное сообщение «Вы отписаны».
15. Повторно открыть ссылку и повторить POST: ошибки и дубля быть не должно.

## Импорт после отписки

1. Создать новую рассылку.
2. Загрузить CSV с `ok@example.test` и ещё одним валидным адресом.
3. Проверить, что `ok@example.test` не попал в accepted recipients.
4. Проверить, что пользователь видит только агрегированный счётчик «Исключены по глобальной отписке».
5. Проверить, что список конкретных отписавшихся адресов пользователю не раскрывается.

## Отписка между импортом и отправкой

1. Создать рассылку и импортировать адрес, который ещё не был отписан.
2. До запуска отправки отписать этот адрес через unsubscribe link из другой fake-рассылки или временно добавить запись в БД.
3. Запустить отправку.
4. Проверить, что для адреса создан `SendEvent` со статусом `Skipped` и причиной `GlobalSuppression`.
5. Проверить, что fake provider не получил это письмо и provider message id отсутствует.

## Postgres persistence

1. Включить `Persistence:Provider=Postgres`.
2. Задать `ConnectionStrings:PismoletDb`.
3. Запустить приложение, чтобы применились migrations.
4. Пройти happy path unsubscribe.
5. Проверить наличие строки в `global_suppressions`.
6. Перезапустить приложение.
7. Загрузить CSV с тем же email.
8. Убедиться, что email всё ещё исключается по глобальной отписке.

## Безопасность публичного flow

1. Изменить один символ в token.
2. Открыть `/unsubscribe/{tamperedToken}`.
3. Убедиться, что страница не раскрывает существование email/рассылки.
4. Проверить, что preview письма в редакторе/модерации не содержит рабочего unsubscribe token.
5. Проверить, что в Sprint 7 не появился новый admin action suppression list, доступный любому авторизованному пользователю.
