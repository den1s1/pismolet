# Sprint 5 — ручные проверки

Ветка: `Development`

## 1. Нормальное письмо

1. Войти в dev-среду.
2. Создать рассылку.
3. Загрузить CSV с валидными адресами.
4. Подтвердить базу.
5. Написать нейтральное информационное письмо:
   - понятный отправитель;
   - без агрессивной рекламы;
   - без подозрительных ссылок;
   - ссылки только с защищённым протоколом.
6. Пройти fake-оплату.
7. Нажать «Проверить перед отправкой».
8. Для клиента без обязательной премодерации убедиться, что статус стал «Одобрено».
9. Для нового клиента с обязательной премодерацией убедиться, что статус стал «На ручной проверке».

## 2. Подозрительная ссылка

1. Создать новую рассылку и пройти шаги до письма.
2. Добавить в текст незащищённую ссылку или ссылку через сокращатель.
3. Пройти fake-оплату.
4. Запустить проверку.
5. Убедиться, что статус стал «На ручной проверке».
6. Убедиться, что пользователь не видит internal rule codes и risk score.

## 3. Admin approve

1. Открыть `/admin/moderation` под авторизованным пользователем.
2. Открыть карточку проверки.
3. Проверить, что видны:
   - тема письма;
   - email клиента;
   - безопасные причины попадания на проверку;
   - текст письма;
   - служебные блоки.
4. Нажать «Одобрить».
5. Убедиться, что карточка показывает «Одобрено».
6. Открыть пользовательский `/mailings/{id}/checks` и убедиться, что пользователь видит «Одобрено».

## 4. Admin reject

1. Создать новую рассылку, которая попадает на ручную проверку.
2. Открыть карточку в `/admin/moderation`.
3. Нажать «Отклонить».
4. Убедиться, что карточка показывает «Отклонено».
5. Открыть пользовательский `/mailings/{id}/checks` и убедиться, что пользователь видит «Отклонено».
6. Убедиться, что переход к отправке не появился.

## 5. Идемпотентность

1. После запуска проверки обновить `/mailings/{id}/checks`.
2. Повторно нажать «Проверить перед отправкой» с `/mailings/{id}/payment`.
3. Убедиться, что не создаётся новая ручная проверка для той же рассылки.
4. Повторно отправить approve/reject для закрытой карточки.
5. Убедиться, что состояние не ломается, а действие фиксируется в логе карточки.

## 6. Ограничения Sprint 5

- Реальная отправка писем не проверяется: это Sprint 6.
- LLM/внешние антиспам API не используются.
- RBAC для `/admin` пока не реализован: доступ закрыт только общей авторизацией, в коде оставлен TODO на роль администратора.
- EF/Postgres persistence для новых моделей модерации в этом срезе не добавлен; in-memory реализация пригодна для dev/unit/web тестов.
