# Sprint 6 — ручные проверки

## Основной happy path

1. Запустить приложение в dev-среде.
2. Зарегистрироваться и подтвердить email через `/dev/fake-mailer`.
3. Создать рассылку.
4. Загрузить CSV с колонкой `email` и несколькими адресами, например:
   - `ok1@example.test`
   - `ok2@example.test`
   - `please-fail@example.test`
5. Подтвердить базу.
6. Сохранить plain text письмо.
7. Открыть `/mailings/{id}/payment` и пройти fake-оплату.
8. Открыть `/mailings/{id}/checks` и запустить проверку.
9. Если рассылка ушла на ручную проверку, открыть `/admin/moderation` и одобрить её.
10. Открыть `/mailings/{id}/send`.
11. Нажать «Запустить отправку».
12. Обновить `/mailings/{id}/send` и проверить summary:
    - принято к отправке;
    - отправлено;
    - ошибки;
    - отписано / исключено;
    - приостановлено по лимиту;
    - ожидает отправки.

## Проверка fake provider

- Адрес без специальных слов должен перейти в «Отправлено» и получить stable provider id внутри `SendEvent`.
- Адрес с `fail` должен перейти в «Ошибка».
- Адрес с `temp` должен перейти в «Ошибка» с временным fake-кодом.
- Повторный refresh `/send` не должен создавать дубли событий.
- Повторный POST `/send/start` для уже запущенной/завершённой рассылки не должен создавать дубли событий.

## Проверка дневного лимита

1. Открыть `/admin/limits`.
2. Установить для клиента маленький дневной лимит, например `1`.
3. Запустить рассылку на несколько адресов.
4. Проверить, что часть адресов получила статус «Приостановлено» и рассылка получила статус «Приостановлено».
5. Увеличить лимит через `/admin/limits`.
6. Открыть `/mailings/{id}/send` и нажать «Продолжить отправку».

## Проверка global suppression

Для текущего dev-slice публичная Sprint 7 unsubscribe-страница ещё не реализована. Если адрес уже есть в `IGlobalSuppressionRepository`, Sprint 6 должен повторно исключить его непосредственно перед отправкой и отразить в summary как «отписано / исключено».

## Известные ограничения текущего dev-slice

- Очередь реализована как `InlineMailingSendQueue`, а не production Hangfire.
- `SendEvent` хранится in-memory; EF/PostgreSQL migration для `send_events` не добавлена.
- Проверка сохранения статистики после рестарта с PostgreSQL пока не проходит без отдельной persistence-доработки.
